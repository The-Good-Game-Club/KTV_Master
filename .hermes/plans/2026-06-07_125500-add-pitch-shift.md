# Plan: Add Pitch Shift (升降Key) to KTV.py

## Goal
Add `--pitch N` flag to KTV.py, allowing the user to raise/lower the audio key by N semitones. e.g. `--pitch -4` for lowering a female song by 4 semitones.

## Current Pipeline (relevant parts)

```
audio_path (WAV)
  ├── Demucs → accomp_path (instrumental) + vocals_path (for alignment)
  └── remux(video_path, accomp_path) → FFmpeg maps video + accompaniment audio → final.mp4
```

## Approach — FFmpeg `rubberband` filter

Use FFmpeg's built-in `librubberband` filter (`rubberband`), which is already compiled in the user's FFmpeg. This filter does **pitch-preserving time-stretching** — meaning it shifts pitch without changing tempo/speed.

### Pitch factor formula

```
factor = 2^(N / 12)
```

Where N = semitones (positive = raise, negative = lower).

Examples:
| N (semitones) | factor   |
|--------------|----------|
| -4           | 0.7937   |
| -3           | 0.8409   |
| -2           | 0.8909   |
| -1           | 0.9439   |
|  0           | 1.0      |
| +1           | 1.0595   |
| +2           | 1.1225   |
| +3           | 1.1892   |
| +4           | 1.2599   |
| +5           | 1.3348   |

### FFmpeg command addition

In the `remux()` function, when `pitch != 0`:

```python
factor = 2 ** (pitch / 12)
cmd += ["-af", f"rubberband=tempo=1.0:pitch={factor}"]
```

This keeps the tempo at 100% and only changes pitch.

### 1. `parse_args()` — add `--pitch` argument

```python
parser.add_argument(
    "--pitch",
    type=int,
    default=0,
    help="Pitch shift in semitones (e.g. -4 to lower 4 semitones, +2 to raise 2). "
         "Uses FFmpeg rubberband filter. 0 = no change.",
)
```

### 2. `remux()` — accept `pitch` parameter, apply FFmpeg filter

```python
def remux(video_path: Path, accomp_path: Path, output_dir: Path,
          subtitle_path: Path | None = None, pitch: int = 0) -> Path:
```

Add pitch shift logic just before the `cmd` list is built:

```python
if pitch != 0:
    # semitone → frequency ratio: 2^(n/12)
    factor = 2 ** (pitch / 12)
    cmd += ["-af", f"rubberband=tempo=1.0:pitch={factor:.6f}"]
    log.info("Pitch shift: %+d semitones (factor=%.4f)", pitch, factor)
```

### 3. `main()` — pass pitch from args to remux

```python
final_path = remux(video_path, accomp_path, args.output.resolve(),
                   subtitle_path=subtitle_path, pitch=args.pitch)
```

### 4. (Optional) `lyrics_only` mode consideration

In `lyrics_only` mode, `accomp_path = audio_path` (original audio including vocals). The pitch shift still works correctly — it shifts the entire track (vocals + instruments). This is actually **desirable** for the common use case: original song re-pitched for easier singing.

## Usage examples

```bash
# Lower A-lin song by 4 semitones for male range
echo "URL" | python KTV.py --mode lyrics_only --pitch -4

# Full karaoke (vocals removed) + lower 3 semitones
echo "URL" | python KTV.py --pitch -3

# Raise by 2 semitones (e.g. male song too low)
echo "URL" | python KTV.py --mode lyrics_only --pitch +2
```

## Risks & edge cases

| Risk | Mitigation |
|------|-----------|
| Extreme pitch shifts (±12+) may sound unnatural | Let user decide — not our job to limit |
| rubberband filter not compiled in FFmpeg | Already confirmed: `--enable-librubberband` is present |
| rubberband adds ~1-2s processing time | Acceptable for a karaoke pipeline |
| Integer semitone may be too coarse | Start with `--pitch int`; can add float later if needed |
| Output filename doesn't reflect pitch change | Could add e.g. `_pitch-4` suffix — low priority |
